## 목표 (Objective)

구현 완료 산출물(`implementation-notes.md` `Status: done` + `kb/artifacts/<id>-summary.md` `Status: done`)을 Codex `review.codex=gpt-5.5/xhigh` 프로필로 검토해 task-local review 문서로 누적한다. `reviews/`가 존재하는 task는 최신 review status가 `approved`일 때만 최종 done gate를 통과하도록 강화하되, reviews가 없는 기존 task는 현재 `--check-done` 동작을 그대로 유지한다.

## 제약 (Constraints)

- 하위호환:
  - `reviews/` 디렉터리가 없는 task는 기존 `--check-done` 결과와 동일해야 한다.
  - `task-001` legacy allowlist는 파일 상태와 무관하게 계속 조기 통과해야 한다.
  - 완료된 기존 task-002~004는 review 디렉터리가 없으므로 새 gate에 영향받지 않아야 한다.
- no-auto-revert:
  - review runner와 validator는 review 파일 외 task status 파일을 변경하지 않는다.
  - `request-changes`, `rejected`, `pending`은 최종 done gate를 실패시킬 뿐, 구현 상태를 자동 변경하지 않는다.
- approved-done:
  - `reviews/`가 없으면 review gate를 적용하지 않는다.
  - `reviews/`가 있으면 `^[0-9]{3}\.md$`에 맞는 최신 review 파일이 필요하다. 디렉터리만 있고 review 파일이 없으면 실패한다.
  - 최신 review 문서가 validator를 통과하지 못하거나 status가 `approved`가 아니면 `--check-done`은 exit `1`로 실패한다.
- codex 쓰기 표면:
  - `codex exec --sandbox workspace-write -C <root> -m gpt-5.5 -c model_reasoning_effort=xhigh`를 항상 명시한다.
  - `--skip-git-repo-check`는 금지한다.
  - Codex prompt는 `review` 출력 파일 하나만 작성하도록 지시하고, design/implementation notes/artifact summary/collab.md를 수정하지 못하도록 명시한다.
- 재귀 가드:
  - nested agent session에서 Codex 자동 호출을 거부하고 `CODEX_AUTO_FORCE=1`에서만 우회한다.
  - `codex-review`는 실행 자체가 review 생성을 목표로 하므로 guard로 실제 review를 만들지 못하면 false success를 내지 않고 non-zero로 종료한다.
- review status는 task status와 별도 상태다.
  - design readiness(`design.md`), implementation progress(`implementation-notes.md`), review verdict(`reviews/NNN.md`)를 한 필드에 섞지 않는다.
- review preflight는 순환 의존을 만들면 안 된다.
  - `codex-review`가 실행 가능해야 하는 조건은 구현 산출물의 base done 상태다.
  - 이전 review가 `request-changes`라서 `--check-done`이 실패하더라도, 수정 구현 후 새 review를 만들 수 있어야 한다. 이를 위해 runner는 `--check-review-target`처럼 review gate를 제외한 target 검증을 사용한다.
- collab.md 과부하 방지:
  - `collab.md`는 enum/게이트/인터페이스 설명만 보유한다.
  - 실제 review 기록과 provenance는 per-task `reviews/NNN.md`에 남긴다.

## 구현 단계 (Implementation Steps)

1. review contract와 template를 정의한다.
   - `collab.md`의 status를 placeholder에서 Phase D active로 갱신한다.
   - enum `pending | request-changes | approved | rejected`, no-auto-revert, approved-done 설명은 계속 `collab.md`에 둔다.
   - 실제 데이터 위치는 `kb/tasks/<id>/reviews/<NNN>.md`임을 명시해 collab이 전역 append 로그가 되지 않게 한다.
   - `templates/review.md`에는 `Reviewer`, `Target`, `Review status`, `Generated by`, `Feedback`, `Action required`를 둔다. `Action required`가 없을 때도 `없음`처럼 명시 값을 쓰게 한다.
2. review validator를 추가한다.
   - `runtime/validator/schema.json`에 review contract path(`collab.md`), filename regex, 필수 필드명, review directory 이름, latest-review 정책을 추가한다.
   - `runtime/validator/review_rules.py`를 신설해 collab enum 파싱, review 필드 추출, status enum 검증, feedback/action-required 비어 있음 검증, latest review 탐색을 담당한다.
   - collab enum 파싱 실패, review 파일 디코딩 실패, schema/config IO 실패는 exit `2` 계열 환경 오류로 취급한다.
   - review 문서 형식 오류나 status enum 위반은 exit `1` 계약 위반으로 취급한다.
3. validator CLI를 확장한다.
   - `--check-review <review-file>`: 단일 review 문서를 검증하고 human/JSON 출력 계약을 제공한다.
   - `--latest-review <task-id-or-dir>`: `reviews/NNN.md` 중 최신 파일과 status를 조회한다. review가 없거나 invalid면 exit `1`, IO/config 오류면 exit `2`.
   - `--check-review-target <task-id-or-dir>`: `implementation-notes.md`, artifact summary, manifest의 base done 조건만 검증한다. review gate는 의도적으로 제외해 재리뷰 순환 차단을 방지한다.
   - 기존 design 검증 경로의 출력과 JSON payload는 변경하지 않는다.
4. `--check-done` approved-done gate를 구현한다.
   - 기존 notes/summary/manifest 검증 결과를 유지한다.
   - `reviews/`가 없으면 여기서 종료해 기존 동작과 동일하게 통과/실패를 결정한다.
   - `reviews/`가 있으면 latest review를 찾아 `--check-review`와 같은 규칙으로 검증한다.
   - latest status가 `approved`가 아니면 `review_not_approved` 오류를 추가한다.
   - 이 과정에서 implementation notes, summary, manifest, review 파일을 수정하지 않는다.
5. review prompt 렌더링을 연결한다.
   - `runtime/render-prompt.py`의 `render` phase choices에 `review`를 추가한다.
   - review prompt substitution에 `TASK_ID`, `DESIGN_FILE`, `IMPL_NOTES`, `ARTIFACT_SUMMARY`, `REVIEW_FILE`, `REVIEW_TEMPLATE`, `PROJECT_ROOT`, `MODEL`, `EFFORT`, `REVIEW_STATUS_ENUM`을 포함한다.
   - `templates/prompts/review.md`는 Codex에게 세 입력 경로를 읽고 review output 파일 하나만 쓰라고 지시한다.
   - final review는 `pending`을 남기지 않는 것을 원칙으로 하되, 도구/입력 문제로 판정 불가한 경우에만 `pending`을 허용한다. 이 경우 done gate는 실패한다.
6. Codex review 호출을 구현한다.
   - `runtime/config/model-profiles.json`의 `review.codex` 예약 표기를 활성 소비자로 갱신한다.
   - `runtime/lib/invoke-codex.{sh,ps1}`에 review용 public function 또는 phase-aware shared helper를 추가한다. 기존 `invoke_codex_if_enabled` design 계약은 보존한다.
   - review 호출은 profile phase `review`, cli `codex`에서 model/effort/min version을 읽는다.
   - git repo preflight, CLI presence, version preflight, sandbox/model/effort flags, `--skip-git-repo-check` 금지를 design 호출과 같은 의미로 적용한다.
   - 호출 성공 시 review 파일 내부 `Generated by` 필드에 `review=codex gpt-5.5/xhigh @codex <version>, <YYYY-MM-DD> (sandbox=workspace-write, target=<NNN>.md)` 형식의 provenance를 남기게 한다.
7. `runtime/codex-review.{sh,ps1}`를 작성한다.
   - 인자: `<task-id>`와 `--help`/`-h`; PowerShell은 `-Help` 또는 기본 help 문구를 제공한다.
   - task dir, design, implementation notes, artifact summary 경로를 계산한다.
   - `validator/cli.py --check-review-target <task-id>`로 base done preflight를 수행한다.
   - next review 번호를 계산한다.
   - Codex가 쓸 임시 review 파일은 counted `reviews/NNN.md` 밖에 둔다. 검증 성공 후에만 `reviews/`를 만들고 `NNN.md`로 move해 partial review가 approved-done gate를 오염시키지 않게 한다.
   - 최종 `NNN.md`를 `--check-review`로 검증한다.
   - 성공 출력에는 review path와 status를 보여준다. status가 `approved`가 아니면 no-auto-revert 원칙과 다음 액션이 사람/구현자 판단임을 안내한다.
8. 문서를 갱신한다.
   - `runtime/README.md`: `codex-review.{sh,ps1}` CLI 계약, Codex review profile, exit code, provenance, `--check-done` review gate를 추가한다.
   - `QUICKREF.md`: Codex 역할에 reviewer를 추가하고, 구현 완료 후 opt-in `runtime/codex-review.sh <id>` 단계를 짧게 안내한다.
   - `AGENT.md`: "리뷰 루프 v2 예약"을 "활성"으로 바꾸고, review data는 per-task reviews에 축적된다고 명시한다.
   - `kb/concepts/workflow.md`: ③ 완료·집계 이후 선택적 ④ Codex 구현 리뷰 단계를 추가한다. 자동 트리거가 아니라 opt-in임을 명시한다.
9. 테스트를 추가·확장한다.
   - validator pytest에 review field/status enum, collab enum parsing, latest review sorting, empty reviews dir, invalid latest review, approved/request-changes done gate 케이스를 추가한다.
   - `--check-done` 회귀 테스트에서 task-001 legacy와 task-002~004 no-review 통과를 유지한다.
   - no-auto-revert 테스트는 `request-changes` review를 둔 뒤 `--check-done`이 실패해도 implementation notes와 summary status가 그대로 `done`인지 확인한다.
   - `tests/run-smoke.sh`에 codex-review runner 스텁 시나리오를 추가한다.
   - `tests/bats/codex-review.bats`와 `tests/pester/codex-review.Tests.ps1`를 추가하고 같은 의미의 시나리오 수를 유지한다.
   - Codex 스텁은 `--sandbox workspace-write`, `-m gpt-5.5`, `model_reasoning_effort=xhigh`, `--skip-git-repo-check` 부재, review numbering/provenance를 검증한다.

## 오픈 이슈 (Open Issues)

- 실제 Codex CLI 인증과 모델 가용성은 단위 테스트에서 증명하지 않는다. 테스트는 스텁으로 wrapper 계약을 고정하고, 로컬 통합 검증은 실제 CLI 환경에서 별도 수행한다.
- Codex가 prompt를 무시하고 review 파일 외 다른 파일을 수정할 가능성은 sandbox만으로 완전히 제거되지 않는다. runner는 prompt로 쓰기 대상을 제한하고, 테스트는 정상 경로를 고정하되, 최종 구현 검증에서는 `git diff -- kb/tasks/<id>/implementation-notes.md kb/artifacts/<id>-summary.md collab.md` 등으로 no-auto-revert를 확인해야 한다.
