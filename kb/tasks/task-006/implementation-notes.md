## 구현 결정 기록

1. **staging → 승격 패턴**: codex 는 `.review-staging.md`(reviews/ 밖)에 쓰고, `--check-review` 통과 시에만
   `reviews/NNN.md` 로 move. 부분/오류 리뷰가 approved-done 게이트를 오염시키지 않게 한다(설계 7단계).
2. **재리뷰 순환 방지**: `--check-review-target`(리뷰 게이트 제외 base 검증)을 러너 전제로 사용. 이전 리뷰가
   request-changes 라 `--check-done` 이 실패해도 새 리뷰를 만들 수 있다.
3. **하위호환**: approved-done 은 `reviews/` 존재 시에만 적용. task-001~005 는 reviews/ 가 없어 기존 done-gate
   동작 유지. legacy task-001 allowlist 도 그대로. (pytest 회귀로 고정)
4. **no-auto-revert**: 러너/validator 는 review 파일 외 어떤 task status 파일도 수정하지 않는다. 최신 리뷰가
   미승인이면 done 게이트만 실패시키고 구현 상태는 그대로 둔다.
5. **codex 재귀 가드**: 리뷰 생성이 목적이므로, 세션 내부에서 막히면 false success 없이 non-zero 로 종료.
6. task-006 자신은 reviews/ 가 없으므로 base done-gate 로 완료 판정(리뷰 루프는 opt-in). self-review dogfood 는
   후속 선택 사항으로 남김.

## 발생한 이슈

- smoke 작성 중 `set -e` 하에서 bare `--check-done`(exit 1 반환)이 스크립트를 조기 종료시킴 → `set +e` 로 감싸
  종료코드를 변수로 캡처하도록 수정.
- 실제 Codex 가 프롬프트를 어기고 다른 파일을 수정할 가능성(설계 오픈이슈)은 sandbox+프롬프트+staging 으로
  완화하되, 최종 통합 검증에서 `git diff` 로 no-auto-revert 를 확인해야 한다(문서화).
