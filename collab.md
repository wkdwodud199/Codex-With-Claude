# collab.md — 협업 리뷰 인터페이스 (v2 active, Phase D)

> **Status: active (Phase D, task-006)**
> 이 문서는 리뷰 루프의 **인터페이스·enum·게이트 정본**이다.
> 실제 리뷰 데이터는 여기 쌓지 않고 per-task `kb/tasks/<id>/reviews/<NNN>.md` 에 저장한다
> (collab.md 가 전역 append 로그가 되는 것을 방지 — per-task 축적 + 파일 과부하 방지).

## 용도

- Codex 가 Claude 의 **구현 결과**(implementation-notes + artifact summary)를 리뷰한다.
- 러너: `runtime/codex-review.{sh,ps1} <task-id>` (opt-in). 산출물: `kb/tasks/<id>/reviews/<NNN>.md`.
- Claude 는 리뷰를 읽고 재구현/수정을 판단한다 (자동 되돌림 없음 — 아래 게이트 참조).

## review status enum (정본)

리뷰 상태(review status)는 task 상태(design 준비도 / 구현 진행도)와 **분리된 별도 필드**다.

<!-- review-status-enum: pending | request-changes | approved | rejected -->

| status | 의미 |
|--------|------|
| `pending` | 판정 보류 (도구/입력 문제로 판정 불가한 경우에만; done 게이트는 실패) |
| `request-changes` | 수정 요청 — 구현자가 반영 후 재리뷰 |
| `approved` | 승인 — approved-done 게이트 통과 |
| `rejected` | 반려 |

> 위 HTML 주석 `review-status-enum:` 라인이 검증기(`runtime/validator/review_rules.py`)가 읽는
> **기계판독 정본**이다. 표는 사람이 읽는 요약이며 주석과 값이 일치해야 한다.

## 리뷰 문서 스키마 (per-task)

`kb/tasks/<id>/reviews/<NNN>.md` (NNN = 3자리 zero-pad, 최댓값+1로 누적). 템플릿: [templates/review.md](./templates/review.md).

```markdown
## Review: task-<NNN> — <YYYY-MM-DD>

- **Reviewer**: Codex
- **Target**: kb/tasks/task-<NNN>/implementation-notes.md
- **Review status**: approved
- **Generated by**: review=codex gpt-5.5/xhigh @codex <ver>, <date> (...)
- **Feedback**:
  - (리뷰 내용)
- **Action required**:
  - (없으면 `없음`)
```

## 게이트 (machine-enforced)

- **no-auto-revert**: `request-changes`/`rejected`/`pending` 리뷰가 나와도 task status
  (implementation-notes / artifact summary 의 `Status`)를 **자동으로 되돌리지 않는다**. 리뷰는
  `reviews/` 에만 기록되고, 다음 액션은 구현자가 판단한다.
- **approved-done**: `kb/tasks/<id>/reviews/` 가 **존재하면** `--check-done` 은 최신 리뷰가
  유효하고 `Review status: approved` 일 때만 통과한다. reviews/ 가 **없으면** 기존 done-gate 동작을
  그대로 유지한다(리뷰 없는 기존 task 하위호환). legacy `task-001` allowlist 는 계속 조기 통과.
- **재리뷰 순환 방지**: 이전 리뷰가 `request-changes` 라 `--check-done` 이 실패해도, 수정 구현 후
  새 리뷰를 만들 수 있어야 한다. 러너는 review 게이트를 제외한 base 검증(`--check-review-target`)으로
  전제조건만 확인한다.
